# A股可比公司分析（Trading Comps）

## ⚠️ 关键：数据源优先级（请先阅读）

**严格遵循以下数据源优先级：**

1. **首选：`tushare-data` skill** — 调用 `daily_basic`、`fina_indicator`、`income`、`stock_basic` 等接口获取行情与财务数据
   - `tushare-data` skill 自动处理 `TUSHARE_TOKEN` 认证
   - `daily_basic` 提供 PE-TTM、PB、总市值、流通市值、换手率等每日估值指标
   - `fina_indicator` 提供 ROE、毛利率、净利率、资产负债率等季度/年度财务质量指标
   - `income` 提供营业收入、营业成本、研发费用等利润表数据
2. **次选：交易所公告与公司定期报告** — 上交所、深交所、北交所官方披露的年报/季报，用于交叉验证或补充 `tushare-data` skill 未覆盖的细项
3. **补充：Web搜索** — 鼓励用于获取行业背景、政策动态、非上市竞争对手数据、行业研究报告摘要（券商研报、艾瑞、易观等）
4. **避免将通用网页搜索作为唯一数据源** — 财务硬数据必须优先通过 `tushare-data` skill 或交易所公告获取，网页信息仅作补充与交叉验证

**原因：** `tushare-data` skill 通过 Tushare 及交易所公告提供经审计的、可追溯的A股数据。Web搜索可用于补充行业语境和定性判断，但不应用作未经核实的财务硬数据的主要来源。

## 概述

本技能指导Agent构建机构级A股可比公司分析，结合经营指标、估值倍数与统计基准。输出为结构化的Excel/电子表格，通过同业比较辅助投资决策。

**参考材料与使用方式：**

使用本技能目录下的示例文件时，请遵循以下原则：

**可用于：**
- 理解结构层次（各板块如何组织）
- 把握严谨度要求（统计深度、文档标准）
- 学习原则（清晰表头、透明公式、审计追溯）

**不可用于：**
- 完全复制格式或指标
- 不考虑上下文直接套用版式
- 忽视受众差异统一视觉风格

**始终先问自己：**
1. **"您是否有偏好的格式，还是由我根据场景适配？"**
2. **"受众是谁？"**（投委会、董事会、快速参考、详细备忘录）
3. **"核心问题是什么？"**（估值、成长性、竞争格局、经营效率）
4. **"场景是什么？"**（并购评估、投资决策、行业对标、绩效回顾）

**根据具体情况调整：**
- **行业背景**：银行保险需要不同于新能源或半导体的指标
- **行业特定需求**：及早加入相关指标（如白酒的预收账款与吨价、新能源的产能利用率、银行的净息差与不良率）
- **公司熟悉度**：知名公司可减少背景介绍，聚焦差异分析
- **决策类型**：并购分析与持续持仓监控侧重点不同

**核心原则：** 使用模板原则（清晰结构、统计严谨、透明公式），但执行方式因场景而异。目标是机构级质量分析，而非机构级模板外观。

用户提供的示例和明确偏好永远优先于默认设置。

## 核心哲学
**"先搭对结构，再让数据讲故事。"**

从强制思考关键问题的表头开始，输入干净数据，构建透明公式，让统计结果自然浮现。一份好的comps应该让没参与制作的人也能立即读懂。

## ⚠️ 关键：公式而非硬编码 + 分步验证

**环境 — Office JS vs Python：**
- **若在Excel内运行（Office Add-in / Office JS）：** 直接使用Office JS（`Excel.run(async (context) => {...})`）。通过 `range.formulas = [["=E7/C7"]]` 写入公式，而非 `range.values`。无需额外重算步骤 — Excel原生处理。使用 `range.format.*` 设置颜色/字体。
- **若生成独立 .xlsx 文件：** 使用 Python/openpyxl。写入 `cell.value = "=E7/C7"`（公式字符串）。
- 两种方式原则相同 — 只是API调用翻译。
- **Office JS合并单元格陷阱：** 不要先 `.merge()` 再对合并区域设置 `.values`（会抛出 `InvalidArgument` — 区域仍报告合并前尺寸）。应先将值写入左上角单元格，再合并+格式化完整区域：
  ```js
  ws.getRange("A1").values = [["白酒行业 — A股可比公司分析"]];
  const hdr = ws.getRange("A1:H1");
  hdr.merge();
  hdr.format.fill.color = "#1F4E79";
  hdr.format.font.color = "#FFFFFF";
  hdr.format.font.bold = true;
  ```

**公式，而非硬编码：**
- 每个派生值（利润率、倍数、统计量）必须是引用输入单元格的Excel公式 — 绝不要粘贴预计算数字
- 使用Python/openpyxl构建时：写入 `cell.value = "=E7/C7"`（公式字符串），而不是 `cell.value = 0.687`（计算结果）
- 唯一允许硬编码的是原始输入数据（营收、净利润、股价等）— 且每个硬编码单元格都要添加批注注明来源
- 原因：模型必须在输入变化时自动更新。硬编码的利润率是潜伏的bug。

**与用户分步验证：**
- 搭建结构后 → 向用户展示表头布局，再填充数据
- 输入原始数据后 → 向用户展示输入区块，确认来源/期间后再构建公式
- 构建经营指标公式后 → 展示计算的利润率，与用户做合理性校验，再进入估值
- 构建估值倍数后 → 展示倍数，确认合理性后再添加统计量
- 不要从头到尾建完再展示 — 在每个阶段尽早发现错误

## 第一节：文档结构与设置

### 表头区块（第1-3行）
```
第1行：[分析标题] — A股可比公司分析
第2行：[公司列表及代码] • [贵州茅台 (600519.SH)] • [五粮液 (000858.SZ)] • [泸州老窖 (000568.SZ)]
第3行：截至 [期间] | 金额单位：人民币 百万元/亿元，每股数据及比率除外
```

**为何重要：** 立即建立上下文。任何打开文件的人都知道在看什么、何时创建、如何解读数字。

### 视觉规范标准（可选 — 用户偏好和上传模板永远优先）

**重要：以下仅为建议默认值。始终优先采用：**
1. 用户明确的格式偏好
2. 任何上传模板文件的格式
3. 公司/团队风格指南
4. 以下默认值（仅在没有其他指导时）

**建议字体与排版：**
- **字体**：Times New Roman（专业、可读、行业标准）
- **字号**：数据11pt，表头12pt
- **加粗**：分节标题、公司名称、统计量标签

**默认颜色与填充 — 专业蓝/灰调色板（极简更好）：**
- **保持克制** — 只用蓝色和灰色。不要引入绿色、橙色、红色或多重强调色。干净的comps表总共只用3-4种颜色。
- **分节标题**（如 "经营指标与财务数据"）：
  - 深蓝背景（`#1F4E79` 或 `#17365D` 藏青色）
  - 白色加粗文字
  - 全行填充跨所有列
- **列标题**（如 "公司"、"营收"、"利润率"）：
  - 浅蓝背景（`#D9E1F2` 或类似淡蓝）
  - 黑色加粗文字
  - 居中对齐
- **数据行**：
  - 白色背景用于公司数据
  - 公式用黑色文字；输入用蓝色文字
- **统计行**（最大值、75分位等）：
  - 浅灰背景（`#F2F2F2`）
  - 黑色文字，左对齐标签
- **这就是整个调色板**：深蓝 + 浅蓝 + 浅灰 + 白色。除非用户模板另有要求，否则不加其他颜色。

**建议格式约定：**
- **小数精度**：
  - 百分比：1位小数（12.3%）
  - 倍数：1位小数（13.5x）
  - 金额：无小数，千分位分隔符（69,632）
  - 利润率以百分比展示：1位小数（68.7%）
- **边框**：无边框（干净、极简外观）
- **对齐**：所有指标居中，整洁统一
- **单元格尺寸**：列宽均匀一致，行高统一（形成干净专业的网格）

**注意：** 若用户提供模板文件或指定不同格式，请按用户要求执行。

## 第二节：经营指标与财务数据

### 核心列（从这里开始）
1. **公司** — 统一格式的公司名称（股票简称）
2. **股票代码** — 6位数字代码（如 600519、000858）
3. **申万/中信行业** — 行业分类（如 食品饮料 — 白酒III / 中信白酒）
4. **营业收入** — 规模指标（通常用TTM或最近年报，人民币百万元/亿元）
5. **营收增速(YoY)** — 同比增速（%）
6. **毛利率** — （营业收入-营业成本）/营业收入
7. **归母净利润** — 归属于母公司股东的净利润
8. **净利率** — 归母净利润/营业收入
9. **ROE** — 归母净利润/归母净资产

### 可选补充（根据行业/目的选择）
- **经营现金流净额** — 经营活动现金流量净额
- **自由现金流** — 经营现金流净额-资本开支（需从 `cashflow` 和 `balancesheet` 推算）
- **研发费用率** — 研发费用/营业收入（科技、医药、新能源制造业必备）
- **扣非净利润 / 扣非净利率** — 剔除非经常性损益后的盈利质量
- **资产负债率** — 总负债/总资产
- **利息保障倍数** — EBIT/利息费用

### 公式示例（以第7行为例）
```excel
// 核心比率 — 始终计算
营收增速 (E7): =([本期营收]-[上期营收])/[上期营收]
毛利率 (F7): =([营业收入]-[营业成本])/[营业收入]
净利率 (H7): =G7/D7
ROE (I7): =G7/[归母净资产]

// 可选比率 — 相关时纳入
研发费用率: =[研发费用]/[营业收入]
经营现金流/净利润: =[经营现金流净额]/[归母净利润]
资产负债率: =[总负债]/[总资产]
```

**黄金法则：** 每个比率应为 [某指标] / [营业收入] 或 [某指标] / [本表另一指标]。保持简单。

### 统计区块（公司数据之后）

**关键：为所有可比指标（比率、利润率、增速、倍数）添加统计公式。**

```
[留一行空白用于视觉分隔]
- 最大值: =MAX(B7:B9)
- 75分位: =QUARTILE(B7:B9,3)
- 中位数: =MEDIAN(B7:B9)
- 25分位: =QUARTILE(B7:B9,1)
- 最小值: =MIN(B7:B9)
```

**需要统计的列（可比指标）：**
- 营收增速%、毛利率%、净利率%、ROE
- 总市值、PE-TTM、PB、PS-TTM、股息率%

**不需要统计的列（规模指标）：**
- 营业收入、归母净利润（绝对规模因公司体量差异大）
- 总市值（不同规模公司不可比，但可作为参考列展示）

**注意：** 公司数据与统计行之间留一行空白。不要添加 "行业统计" 或 "估值统计" 标题行。

**为何分位重要：** 它们展示分布，而非仅平均值。75分位倍数告诉你"优质"公司的交易水平。

## 第三节：估值倍数与投资指标

### 核心估值列（从这里开始）
1. **公司** — 与经营部分顺序一致
2. **总市值** — 当前总股本 × 收盘价（人民币亿元）
3. **PE-TTM** — 市盈率（滚动十二个月），A股最常用的估值指标
4. **PB** — 市净率（最新财报）
5. **PS-TTM** — 市销率（总市值/营业收入-TTM）
6. **股息率** — 年度分红/股价

### 可选估值指标（根据场景选择）
- **PEG** — PE/净利润增速（成长股适用）
- **流通市值** — 流通股本 × 收盘价
- **EV/EBITDA** — A股实践中较少直接使用，需手动从 `balancesheet` 计算净负债后推导。提及即可，不要求必须纳入
- **股息支付率** — 分红金额/净利润
- **营收CAGR** — 历史复合增长率
- **净利润CAGR** — 历史复合增长率

**关键原则：** A股市场通常最关注PE-TTM和PB。纳入3-5个对行业至关重要的核心倍数即可，不要因为有就全部罗列。EV-based倍数在A股研究中相对少见，如需使用请明确说明计算过程（EV = 总市值 + 有息负债 - 货币资金）。

### 公式示例
```excel
// 核心倍数 — 始终纳入
PE-TTM: =[总市值]/[归母净利润-TTM]
PB: =[总市值]/[归母净资产]
PS-TTM: =[总市值]/[营业收入-TTM]
股息率: =[每股分红]*[总股本]/[总市值]

// 可选倍数 — 数据可得时纳入
PEG: =[PE-TTM]/[净利润增速%]
```

### 交叉引用规则
**关键：** 估值倍数必须引用经营指标部分的数据。同一原始数据不要输入两次。若营业收入在D7，则PS-TTM公式应引用D7。

### 统计区块
与经营部分结构相同：最大值、75分位、中位数、25分位、最小值。公司数据与统计之间留一行空白。不要添加 "估值统计" 标题行。

## 第四节：附注与方法论文档

### 必需内容

**数据来源与质量：**
- 数据来源？（`tushare-data` skill 调用 `daily_basic`/`fina_indicator`/`income` 接口、交易所年报、Web搜索-行业研究报告）
- 覆盖期间？（2024年报、TTM数据）
- 如何验证？（与交易所披露 `anns_d` 或巨潮资讯网交叉核对）
- 注：优先通过 `tushare-data` skill 获取结构化数据，确保准确性与可追溯性；Web搜索仅用于补充行业语境

**关键定义：**
- EBITDA计算方法（营业收入-营业成本-税金及附加-销售/管理/研发费用+折旧摊销，或营业利润+折旧摊销）
- 自由现金流公式（经营活动现金流净额-资本开支）
- 特殊指标说明（扣非净利润、PE-TTM、PS-TTM）
- 时间定义（TTM、LF、CAGR计算期间）
- 申万/中信行业分类版本说明

**估值方法论：**
- 企业价值如何计算？（总市值+净负债，净负债=有息负债-货币资金）
- 使用什么增速？（历史CAGR、一致预期增速）
- 做了哪些调整？（剔除非经常性损益、季节性因素）

**分析框架：**
- 投资逻辑是什么？（行业景气度、竞争格局、政策环境）
- 哪些指标最重要？（盈利质量、成长性、估值安全边际）
- 读者应如何解读统计量？（分位数提供估值锚点）

## 第五节：选择合适的指标（决策框架）

### 从"我在回答什么问题？"开始

**"哪家公司被低估了？"**
→ 聚焦：PE-TTM、PB、PS-TTM、总市值、股息率
→ 跳过：经营细节、成长性指标

**"哪家公司效率最高？"**
→ 聚焦：毛利率、净利率、ROE、资产周转率
→ 跳过：规模指标、绝对金额

**"哪家公司成长性最好？"**
→ 聚焦：营收增速%、净利润增速%、CAGR
→ 跳过：利润率指标、杠杆比率

**"哪家公司现金创造能力最强？"**
→ 聚焦：经营现金流净额、经营现金流/净利润、资本开支强度
→ 跳过：EBITDA、PE比率

### 行业特定指标选择

**白酒（如 贵州茅台 600519.SH、五粮液 000858.SZ、泸州老窖 000568.SZ）：**
必备：毛利率、净利率、ROE、预收账款/营业收入
可选：吨价、产能、销售费用率
跳过：研发费用率、资产周转率（轻资产模式）

**新能源/制造业（如 宁德时代 300750.SZ、比亚迪 002594.SZ、隆基绿能 601012.SH）：**
必备：毛利率、净利率、ROE、研发费用率
可选：产能利用率、ROIC、海外收入占比
跳过：PEG（周期性强时参考性弱）

**银行（如 招商银行 600036.SH、平安银行 000001.SZ）：**
必备：ROE、净息差、不良率、PB
可选：拨备覆盖率、核心一级资本充足率
跳过：PE（盈利波动大时参考性弱）、毛利率、EBITDA

**医药（如 恒瑞医药 600276.SH、药明康德 603259.SH）：**
必备：毛利率、研发费用率、ROE
可选：管线价值、核心品种收入占比
跳过：PB（轻资产模式参考性弱）

**SaaS/软件（如 金山办公 688111.SH、用友网络 600588.SH）：**
必备：营收增速、毛利率、预收款/合同负债
可选：研发费用率、PS-TTM
跳过：PE（盈利波动期）、资产周转率

### "5-10法则"

**5个经营指标** — 营收、增速、2-3个利润率/效率指标
**5个估值指标** — 总市值、PE-TTM、PB、PS-TTM、股息率
**= 10个总列** — 足够讲故事，又不至于迷失

如果超过15个指标，可能纳入了噪音。要无情删减。

## 第六节：最佳实践与质量检查

### 开始前
1. **定义可比公司池** — 公司必须真正可比（相似业务模式、规模、地域）
2. **选对期间** — TTM平滑季节性；季度数据展示趋势
3. **统一单位** — 百万元 vs 亿元的决定影响一切
4. **标注数据来源** — 知道每个数字来自哪里

### 构建中
1. **先输入所有原始数据** — 完成蓝色文字输入后再写公式
2. **为所有硬编码输入添加单元格批注** — 右键单元格 → 插入批注 → 记录来源或假设

   **对于来源数据，精确注明出处：**
   - 示例："Tushare daily_basic 接口，2025-05-06，收盘价"
   - 示例："2024年年报，合并利润表，'营业收入' 科目"
   - 示例："Tushare fina_indicator 接口，2024年报，ROE指标"
   - **尽可能添加超链接**：右键单元格 → 链接 → 粘贴交易所公告或数据源URL

   **对于假设，解释推理过程：**
   - 示例："假设毛利率75%，基于同业中位数"
   - 示例："企业价值按总市值+50亿净负债估算（来自2024年三季报）"
   - 示例："forward PE基于一致预期EPS 3.45元（12家券商预测均值）"

   **为何重要**：实现审计追溯、数据验证、假设透明、未来更新便利
3. **逐行构建公式** — 每写一行计算就测试一行
4. **表头使用绝对引用** — $C$6 锁定表头行
5. **格式一致** — 百分比就显示为百分比，不要显示小数
6. **添加条件格式** — 自动高亮异常值

### 合理性检查
- **利润率测试**：毛利率 > 净利率（按定义恒成立）；ROE通常 > ROA
- **倍数合理性**：
  - PE-TTM：A股成长股通常 15-60x，银行常在 <10x，白酒龙头 20-40x
  - PB：通常 0.5-10x（银行常在1x以下，科技/消费可超10x）
  - PS-TTM：通常 1-20x（行业差异大）
  - 股息率：A股高股息板块通常 3-8%
- **增速-倍数相关性**：更高增速通常对应更高估值倍数
- **规模-效率权衡**：更大公司通常利润率更好（规模效应），但增速放缓

### 常见错误
❌ 总市值和流通市值在公式中混用
❌ 分子分母使用不同时间区间（TTM vs 年报）
❌ 把数字硬编码进公式而非使用单元格引用
❌ **硬编码输入没有批注注明来源或解释假设**
❌ 没有添加交易所公告或数据源超链接
❌ 纳入过多无明确目的的指标
❌ 纳入不可比公司（业务模式差异大）
❌ 使用过时数据未披露
❌ 错误地计算百分比平均值（应使用中位数）

### 分位分析优势
不只用均值/中位数，分位数展示：
- **75分位** = "优质"公司的交易/经营水平
- **中位数** = 市场典型水平
- **25分位** = "折价"区间

这帮助回答："目标公司相对于同业是贵还是便宜？"

## 第八节：工作流程与实用技巧

### 分步流程
1. **搭建结构**（30分钟）
   - 创建所有表头
   - 设置单元格格式（蓝色=输入，黑色=公式）
   - 锁定单位和日期引用

2. **收集数据**（60-90分钟）
   - 通过 `tushare-data` skill 拉取（`daily_basic`、`fina_indicator`、`income`、`stock_basic`）
   - Web搜索补充行业语境与竞争对手信息
   - 输入所有原始数字（蓝色）
   - 在附注区记录数据来源

3. **构建公式**（30分钟）
   - 从简单比率开始（利润率）
   - 进阶到倍数（PE-TTM、PB、PS-TTM）
   - 添加交叉校验（利润率合理吗？）

4. **添加统计量**（15分钟）
   - 复制公式结构到所有列
   - 验证范围正确（B7:B9，不是B7:B10）
   - 检查分位逻辑

5. **质量控制**（30分钟）
   - 运行合理性检查
   - 验证公式引用
   - 检查 #DIV/0! 或 #REF! 错误
   - 与已知基准比较

6. **文档编写**（15分钟）
   - 完善附注区
   - 添加数据来源
   - 定义方法论
   - 给分析加日期戳

### 专业提示
- **保存模板**：一次构建，永久复用
- **标记异常值**：对超出2个标准差的值做条件格式
- **链接到源文件**：超链接到 `tushare-data` skill 输出或交易所公告
- **版本控制**：保存为 "Comps_v1_2025-05-06" 并清晰标注日期
- **协作审核**：让别人检查你的公式

## 第十节：红旗与警示信号

### 数据质量问题
🚩 时间区间不一致（混用季报和年报）
🚩 缺失数据未加说明
🚩 不同数据源间存在显著差异（>10%偏差）

### 估值红旗
🚩 负净利润公司用PE估值（改用PS或PB）
🚩 PE-TTM > 100x 且缺乏超高增长逻辑
🚩 利润率对行业明显不合理

### 可比性问题
🚩 不同财年截止日（造成时间错配）
🚩 纯业务公司与多元化集团混用
🚩 业务模式差异大却标为"可比"

**有疑虑时，排除该公司。** 3个完美可比公司胜过6个可疑可比公司。

## 第十一节：公式参考指南

### 核心Excel公式
```excel
// 统计函数
=AVERAGE(range)          // 简单平均
=MEDIAN(range)           // 中位数
=QUARTILE(range, 1)      // 25分位
=QUARTILE(range, 3)      // 75分位
=MAX(range)              // 最大值
=MIN(range)              // 最小值
=STDEV.P(range)          // 标准差

// 财务计算
=B7/C7                   // 简单比率（利润率）
=SUM(B7:B9)/3            // 多公司平均
=IF(B7>0, C7/B7, "N/A")  // 条件计算
=IFERROR(C7/D7, 0)       // 处理除零错误

// 跨表引用
='Sheet1'!B7             // 引用另一工作表
=VLOOKUP(A7, Table1, 2)  // 从数据表查找
=INDEX(MATCH())          // 高级查找

// 格式化
=TEXT(B7, "0.0%")        // 格式化为百分比
=TEXT(C7, "#,##0")       // 千分位分隔符
```

### 常见比率公式
```excel
毛利率 = （营业收入 - 营业成本）/ 营业收入
净利率 = 归母净利润 / 营业收入
ROE = 归母净利润 / 归母净资产
ROA = 归母净利润 / 总资产
资产周转率 = 营业收入 / 总资产
资产负债率 = 总负债 / 总资产
利息保障倍数 = EBIT / 利息费用
```

## 关键原则总结

1. **结构驱动洞察** — 正确的表头迫使正确的思考
2. **少即是多** — 5-10个重要指标胜过20个无关指标
3. **为问题选指标** — 估值分析 ≠ 效率分析
4. **统计展示规律** — 中位数/分位数比平均值揭示更多信息
5. **透明胜过复杂** — 简单公式人人能懂
6. **可比性为王** — 宁可排除也不要强行可比
7. **记录你的选择** — 在附注区解释用了哪些指标及原因
