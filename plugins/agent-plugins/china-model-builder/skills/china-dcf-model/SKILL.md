# A股DCF模型构建器

## 概述

本技能创建符合投行标准的A股DCF估值模型。每次分析生成一份详细的Excel模型（DCF工作表底部包含敏感性分析）。

## 工具

- 默认使用用户提供的信息和可用的Tushare数据接口获取数据。

## 关键约束 — 请先阅读

以下约束适用于所有DCF模型构建过程，开始前请审阅：

**环境：Office JS vs Python/openpyxl：**
- **若在Excel内运行（Office Add-in / Office JS环境）：** 直接使用Office JS — 不要用Python/openpyxl。通过 `range.formulas = [["=D19*(1+$B$8)"]]` 写入公式。无需额外重算步骤；Excel原生计算。使用 `range.format.*` 设置样式。以下原则（公式字符串、单元格批注、分节检查点、敏感性表格循环）均适用，只是API翻译。
- **若生成独立 .xlsx 文件（无实时Excel会话）：** 使用Python/openpyxl，然后运行 `recalc.py` 再交付。
- 其余部分以openpyxl示例描述 — 在Office JS环境中翻译为对应API调用，但所有原则（公式、批注、检查点、敏感性循环）完全一致。

**⚠️ Office JS合并单元格陷阱：** 构建分节标题时，不要先调用 `.merge()` 再对合并区域设置 `.values` — Office JS仍报告区域的原始尺寸并会抛出 `InvalidArgument: The number of rows or columns in the input array doesn't match the size or dimensions of the range`。应先将值写入左上角单元格，再合并和格式化完整区域：

```js
// 错误 — 抛出 InvalidArgument：
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.values = [["市场数据与关键输入"]];  // 1×1 数组 vs 1×8 区域 → 失败

// 正确 — 先对单个单元格写入值，再合并+格式化区域：
ws.getRange("A7").values = [["市场数据与关键输入"]];
const hdr = ws.getRange("A7:H7");
hdr.merge();
hdr.format.fill.color = "#1F4E79";
hdr.format.font.bold = true;
hdr.format.font.color = "#FFFFFF";
```

这适用于DCF中每个合并的分节标题（市场数据、情景区块、现金流预测、终值、估值摘要、敏感性表格）。

**公式优于硬编码（不可妥协）：**
- 每个预测、利润率、折现因子、现值和敏感性单元格必须是实时Excel公式 — 绝不要在Python中计算后写入数字
- 使用openpyxl时：`ws["D20"] = "=D19*(1+$B$8)"` 正确；`ws["D20"] = calculated_revenue` 错误
- 唯一允许硬编码的数字是：(1) 原始历史输入，(2) 假设驱动因素（增速、WACC输入、终值g），(3) 当前市场数据（股价、债务余额）
- 如果你发现自己在Python中计算并写入结果 — 停止。模型必须在用户修改假设时自动调整。

**与用户分步验证（不要从头到尾构建）：**
- 数据获取后 → 向用户展示原始输入区块（营收、利润率、股本、净负债）并确认后再预测
- 营收预测后 → 展示预测顶线及增速，确认后再构建利润率
- FCF构建后 → 展示完整FCF表，确认逻辑后再计算WACC
- WACC后 → 展示计算及输入，确认后再折现
- 终值+现值后 → 展示权益桥（EV → 权益价值 → 每股），确认后再做敏感性表格
- 在每个阶段捕获错误 — 敏感性表格建完后才发现利润率假设错误意味着下游全部重建

**敏感性表格：**
- **使用奇数行和奇数列**（标准：5×5，有时7×7）— 这保证存在真正的中心单元格
- **中心单元格 = 基础情形。** 构建坐标轴值使中间行表头和中间列表头精确等于模型实际假设（例如，基础WACC=9.0%，则中间行为9.0%；终值g=3.0%，则中间列为3.0%）。因此中心单元格的输出必须等于模型实际隐含股价 — 这是表格构建正确的合理性检查。
- **高亮中心单元格** 使用中蓝填充（`#BDD7EE`）+ 加粗字体，使基础情形一目了然。
- 用openpyxl循环（或Office JS循环）程序化写入公式填充所有单元格
- 不要占位文本，不要线性近似，不要手动步骤
- 每个单元格必须对该假设组合重新计算完整DCF

**单元格批注：**
- 每个硬编码值创建时即添加批注
- 格式："来源：[系统/文档]，[日期]，[引用]，[URL如适用]"
- 每个蓝色输入在移入下一节前必须有批注
- 不要推迟到末尾或写"TODO: 添加来源"

**模型布局规划：**
- 写任何公式前先定义所有分节行位置
- 先写所有表头和标签
- 再写所有分节分隔线和空白行
- **然后**用锁定的行位置写公式
- 创建公式后立即测试

**公式重算：**
- 交付前运行 `python recalc.py model.xlsx 30`
- 修复所有错误直到状态为 "success"
- 要求零公式错误（#REF!、#DIV/0!、#VALUE! 等）

**情景区块：**
- 为悲观/基础/乐观情形创建独立区块
- 在每个区块内横向展示预测年份的假设
- 使用IF公式：`=IF($B$6=1,[悲观单元格],IF($B$6=2,[基础单元格],[乐观单元格]))`
- 验证公式引用正确情景区块单元格

## DCF流程工作流

### 第一步：数据获取与验证

从Tushare接口、用户提供的数据和网页获取数据。

**数据源优先级：**
1. **Tushare数据接口**（如配置）— 结构化A股财务数据（income、balancesheet、cashflow、daily_basic、shibor_lpr）
2. **用户提供的数据** — 来自其研究的历史财务数据
3. **网页搜索/获取** — 当前股价、Beta、债务和现金（需要时）

**验证检查清单：**
- 验证净负债 vs 净现金（估值关键）
- 确认总股本（含限售股/流通股本区分，检查近期回购/增发）
- 验证历史利润率与业务模式一致
- 交叉核对营收增速与行业基准
- 验证税率合理（A股通常15%-25%，高新技术企业15%）

### 第二步：历史分析（3-5年）

分析并记录：
- **营收增长趋势**：计算CAGR，识别驱动因素
- **利润率演进**：跟踪毛利率、EBIT利润率、FCF利润率
- **资本密集度**：折旧摊销和资本开支占营收比
- **营运资本效率**：营运资本变动占营收增长比
- **回报指标**：ROIC、ROE趋势

创建汇总表展示：
```
历史指标（TTM）：
营业收入：XX 亿元
营收增速：XX% CAGR
毛利率：XX%
EBIT利润率：XX%
折旧摊销占营收：XX%
资本开支占营收：XX%
FCF利润率：XX%
```

### 第三步：构建营收预测

**方法论：**
1. 从最新实际营收（TTM或最近财年）开始
2. 对每一年预测应用增速
3. 同时展示金额和计算增速%

**增速框架：**
- 第1-2年：较高增速，反映近期可见性
- 第3-4年：逐步向行业均值回落
- 第5年+：接近终值增速

**公式结构：**
- 营收(第N年) = 营收(第N-1年) × (1 + 增速)
- 增速%(第N年) = 营收(第N年) / 营收(第N-1年) - 1

**三情景法：**
```
悲观情形：保守增速（例如 5-8%）
基础情形：最可能情景（例如 8-12%）
乐观情形：乐观增速（例如 12-18%）
```

### 第四步：运营费用建模

**固定/变动成本分析：**

运营费用应建模真实经营杠杆：
- **销售费用**：通常占营收5-30%，取决于商业模式
- **管理费用**：通常占营收5-15%，规模扩大时显示杠杆
- **研发费用**：科技公司通常占营收5-20%

**关键原则：**
- 所有百分比基于**营业收入**，而非毛利
- 建模经营杠杆：百分比随营收规模扩大应下降
- 销售/管理/研发保持独立行项目
- EBIT = 毛利润 - 总运营费用

**利润率扩张框架：**
```
当前状态 → 目标状态（第5年）
毛利率：XX% → YY%（基于规模、效率论证）
EBIT利润率：XX% → YY%（营收增长+运营杠杆的结果）
```

### 第五步：自由现金流计算

**按正确顺序构建FCF：**

```
EBIT
(-) 所得税（EBIT × 税率）
= NOPAT（税后净营业利润）
(+) 折旧摊销（非现金费用，占营收比）
(-) 资本开支（占营收比，通常3-10%）
(-) Δ 营运资本（营运资本变动）
= 无杠杆自由现金流（Unlevered FCF）
```

**营运资本建模：**
- 按营收变动额的百分比计算
- 典型范围：营收变动的 -2% 到 +2%
- 负数 = 现金来源（营运资本释放）
- 正数 = 现金使用（营运资本增加）

**维护性 vs 增长性资本开支：**
- 维护性资本开支：维持当前运营（~营收的2-3%）
- 增长性资本开支：支持扩张（额外2-5%营收）
- 总资本开支应与公司增长战略一致

### 第六步：资本成本（WACC）研究

**CAPM方法论计算权益成本：**

```
权益成本 = 无风险利率 + Beta × 市场风险溢价

其中：
- 无风险利率 = 中国10年期国债收益率（Tushare `shibor_lpr` 或 `us_tycr`）
  *注：若数据不可得，可采用固定假设 3.0% 作为近似*
- Beta = 5年月度股票Beta vs 沪深300/中证500
  *注：若无法获取，可用行业平均 1.0 简化处理*
- 市场风险溢价 = 6.0%-7.0%（中国市场常用值）
```

**债务成本计算：**

```
税后债务成本 = 税前债务成本 × (1 - 税率)

税前债务成本来源：
- 信用评级（如有）
- 公司债券当前收益率
- 利息费用 / 有息负债（来自财务报表）
```

**资本结构权重：**

```
权益市场价值 = 当前股价 × 总股本
净负债 = 有息负债 - 货币资金
企业价值 = 总市值 + 净负债

权益权重 = 总市值 / 企业价值
债务权重 = 净负债 / 企业价值

WACC = (权益成本 × 权益权重) + (税后债务成本 × 债务权重)
```

**特殊情况：**
- **净现金头寸**：若现金 > 有息负债，净负债为负
  - 债务权重可能为负
  - WACC计算相应调整
- **无负债**：WACC = 权益成本

**典型WACC区间（A股）：**
- 大盘蓝筹、稳定型：7-9%
- 成长型公司：9-12%
- 高增长/高风险：12-15%

### 第七步：折现率应用（5-10年预测）

**年中惯例：**
- 假设现金流发生在年中
- 折现期：0.5, 1.5, 2.5, 3.5, 4.5 等
- 折现因子 = 1 / (1 + WACC)^期间

**现值计算：**
```
对每个预测年：
FCF现值 = 无杠杆FCF × 折现因子

示例（第1年）：
FCF = 1,000
WACC = 10%
期间 = 0.5
折现因子 = 1 / (1.10)^0.5 = 0.9535
现值 = 1,000 × 0.9535 = 954
```

**预测期选择：**
- **5年**：大多数分析的标准
- **7-10年**：高增长、长跑道的公司
- **3年**：成熟、稳定的企业

### 第八步：终值计算

**永续增长法（首选）：**

```
终值FCF = 最终年FCF × (1 + 终值增长率)
终值 = 终值FCF / (WACC - 终值增长率)

关键约束：终值增长率 < WACC（否则价值无穷大）
```

**终值增长率选择：**
- 保守：2.5-3.0%（接近中国GDP长期增速下限）
- 中性：3.0-3.5%
- 积极：3.5-4.0%（仅行业龙头适用）

**不要超过：** 无风险利率或中国长期GDP增速（3-4%）

**退出倍数法（替代）：**
```
终值 = 最终年EBITDA × 退出倍数

退出倍数来源：
- A股行业可比交易倍数
- 历史交易倍数
- 典型区间：8-15x EBITDA
```

**终值现值：**
```
终值现值 = 终值 / (1 + WACC)^最终期间

最终期间考虑时点：
5年模型 + 年中惯例：期间 = 4.5
```

**终值合理性检查：**
- 应占企业价值的50-70%
- 若 >75%，模型可能过度依赖终值假设
- 若 <40%，检查终值假设是否过于保守

### 第九步：企业价值到权益价值桥接

**估值摘要结构：**

```
(+) 预测FCF现值之和 = XX 亿元
(+) 终值现值 = YY 亿元
= 企业价值 = ZZ 亿元

(-) 净负债 [或 + 净现金（若为负）] = AA 亿元
= 权益价值 = BB 亿元

÷ 总股本（亿股） = CC 亿股
= 隐含每股价值 = XX.XX 元

当前股价 = YY.YY 元
隐含涨跌幅 = (隐含价 / 当前价) - 1 = XX%
```

**关键调整：**
- **净负债 = 有息负债 - 货币资金**
  - 若为正：从EV中扣除（降低权益价值）
  - 若为负（净现金）：加入EV（提升权益价值）
- **使用总股本**：含限售股；注意区分流通股本与总股本
- **其他调整**（如适用）：
  - 少数股东权益
  - 长期股权投资
  - 经营租赁义务

**估值输出格式：**
```csv
估值组成,金额（亿元）
预测FCF现值之和,X.X
终值现值,Y.Y
企业价值,Z.Z
(-) 净负债,A.A
权益价值,B.B
,,
总股本（亿股）,C.C
隐含每股价值,XX.XX 元
当前股价,YY.YY 元
隐含涨跌幅,+XX%
```

### 第十步：敏感性分析

在DCF工作表底部构建**三张敏感性表格**，展示估值如何随假设变化：

1. **WACC vs 终值增长率** — 展示企业价值对折现率和永续增长的敏感性
2. **营收增速 vs EBIT利润率** — 展示顶线增长和经营杠杆的影响
3. **Beta vs 无风险利率** — 展示权益成本组成部分的敏感性

**实现方式**：这些是简单的二维网格（**不是**Excel的"数据表"功能），每个单元格一个公式。每个单元格必须包含对该特定假设组合的完整DCF重新计算。详见关键约束中关于用openpyxl程序化填充全部75个单元格的详细要求。

### 情景区块选择模式 — 采用此方法

**假设按情景组织为独立区块：**

**关键结构 — 每个分节标题三行：**

```csv
悲观情形假设（分节标题，跨列合并单元格）
假设,FY1,FY2,FY3,FY4,FY5
营收增速 (%),8%,7%,6%,5%,4%
EBIT利润率 (%),15%,14%,13%,12%,11%

基础情形假设（分节标题，跨列合并单元格）
假设,FY1,FY2,FY3,FY4,FY5
营收增速 (%),12%,10%,9%,8%,7%
EBIT利润率 (%),18%,19%,20%,21%,22%

乐观情形假设（分节标题，跨列合并单元格）
假设,FY1,FY2,FY3,FY4,FY5
营收增速 (%),16%,14%,12%,10%,8%
EBIT利润率 (%),20%,21%,22%,23%,24%
```

**每个情景区块必须有一列表头行**展示预测年份（FY2025E、FY2026E等），紧跟在标题下方。没有此行，用户无法知道哪个假设值对应哪一年。

**如何引用假设 — 创建合并列：**
1. 情景选择单元格（如B6）包含1=悲观、2=基础、3=乐观
2. 创建合并列，用INDEX或OFFSET公式从正确情景区块提取
3. 预测公式引用合并列（干净的单元格引用）
4. 每个情景区块包含横跨预测年份的完整DCF假设集

**推荐的合并列模式（使用INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`

**不要这样做 — 分散的IF语句贯穿全文：**
`=IF($B$6=1,[悲观区块单元格],IF($B$6=2,[基础区块单元格],[乐观区块单元格]))`

合并列方法集中逻辑，使模型更易于审计。

### 正确营收预测模式

**创建带INDEX公式的合并列，然后在预测中引用：**

**第一步 — FY1增速合并列：**
`=INDEX([悲观FY1增速]:[乐观FY1增速], 1, $B$6)`

**第二步 — 营收预测引用合并列：**
`营收第1年: =D29*(1+$E$10)`

其中：
- D29 = 上一年营收
- $E$10 = FY1增速合并列单元格（含INDEX公式）
- $B$6 = 情景选择器（1=悲观、2=基础、3=乐观）

**此方法比在每个预测公式中嵌入IF语句更清洁**，且更易于审计使用了哪套情景假设。

### 正确FCF公式模式

**使用带INDEX公式的合并列，然后在FCF计算中引用：**

**合并列方法：**
```csv
项目,公式,引用
折旧摊销,=E29*$E$21,$E$21 = 折旧摊销%合并列
资本开支,=E29*$E$22,$E$22 = 资本开支%合并列
Δ 营运资本,=(E29-D29)*$E$23,$E$23 = 营运资本%合并列
无杠杆FCF,=E57+E58-E60-E62,E57=NOPAT E58=折旧摊销 E60=资本开支 E62=Δ营运资本
```

**每个合并列单元格包含INDEX公式**，根据情景选择器从相应情景区块提取。这保持预测公式清洁且可审计。

编写公式前，确认情景区块行位置并设置合并列。

### 正确单元格批注格式

**每个硬编码值需要此格式：**

"来源：[系统/文档]，[日期]，[引用]，[URL如适用]"

**示例：**
```csv
项目,来源批注
股价,来源：Tushare daily_basic 2025-05-06 收盘价
总股本,来源：2024年年报 第45页 注释12
历史营收,来源：2024年年报 第32页 合并利润表
Beta,来源：Tushare 2025-05-06 5年月度Beta vs 沪深300
一致预期,来源：管理层指引 2024年三季度业绩说明会
```

### 正确假设表结构

**关键：每个情景区块需要三个结构要素：**

1. **分节标题行**（合并单元格）：如 "悲观情形假设"
2. **展示年份的列表头行** — 这是必需的，不可跳过
3. **数据行**，含假设值

**结构：**
```csv
悲观情形假设（分节标题 — 合并A:G列）
假设,FY1,FY2,FY3,FY4,FY5
营收增速 (%),X%,X%,X%,X%,X%
EBIT利润率 (%),X%,X%,X%,X%,X%

基础情形假设（分节标题 — 合并A:G列）
假设,FY1,FY2,FY3,FY4,FY5
营收增速 (%),X%,X%,X%,X%,X%
EBIT利润率 (%),X%,X%,X%,X%,X%
终值增长率,X%,,,,

乐观情形假设（分节标题 — 合并A:G列）
假设,FY1,FY2,FY3,FY4,FY5
营收增速 (%),X%,X%,X%,X%,X%
EBIT利润率 (%),X%,X%,X%,X%,X%
终值增长率,X%,,,,
WACC,X%,,,,
```

**没有展示预测年份（FY2025E、FY2026E等）的列表头行，用户无法知道哪个假设值对应哪一年。此行是强制性的。**

**然后创建合并列**（通常在右侧下一列），使用INDEX公式根据情景选择器从选定情景区块提取。此合并列是你的预测公式所引用的。

### 正确敏感性表格实现

**重要**：这些**不是**Excel的"数据表"功能。这些是简单网格，你用openpyxl写入常规公式。是的，这意味着约75个公式总计（3表×25单元格），但这很直接且是必需的。

**用公式程序化填充：**

每个敏感性表格必须用公式完全填充，对每个假设组合重新计算隐含股价。**不要使用Excel的数据表功能**（它需要手动干预且无法通过openpyxl自动化）。

**实现方式 — 具体示例：**

**表格结构 — 5×5网格（奇数维度，基础情形居中）：**

若模型基础WACC = 9.0%，基础终值增长 = 3.0%，围绕这些值对称构建坐标轴：

```csv
WACC vs 终值增长率,  2.0%,  2.5%,  3.0%,  3.5%,  4.0%
              8.0%, [fml], [fml], [fml], [fml], [fml]
              8.5%, [fml], [fml], [fml], [fml], [fml]
              9.0%, [fml], [fml], [★  ], [fml], [fml]   ← 中间行 = 基础WACC
              9.5%, [fml], [fml], [fml], [fml], [fml]
             10.0%, [fml], [fml], [fml], [fml], [fml]
                                   ↑
                          中间列 = 基础终值g
```

**★ = 中心单元格。** 其公式输出必须等于模型实际隐含股价（来自估值摘要）。对此单元格应用中蓝填充（`#BDD7EE`）和加粗字体，使基础情形视觉锚定。

**坐标轴值规则：** `axis_values = [base - 2*step, base - step, base, base + step, base + 2*step]` — 围绕基础对称，奇数保证中心。

**公式模式 — 单元格B88（WACC=8.0%，终值增长=2.0%）：**

B88的公式应使用以下值重新计算隐含价：
- 行表头的WACC：`$A88`（8.0%）
- 列表头的终值增长：`B$87`（2.0%）

**推荐方法：** 引用主DCF计算但替换这些值。

**示例公式结构：**
`=([使用$A88作为折现率的PV FCF之和] + [使用B$87作为增长率和$A88作为WACC的终值] - [净负债]) / [总股本]`

**关键 — 为5x5网格的每个单元格写公式（每表25单元格，共75单元格）。** 使用openpyxl以循环程序化写入这些公式。不要跳过此步骤或留占位文本。

**Python实现模式：**
```python
# 填充敏感性表格的伪代码
for row_idx, wacc_value in enumerate(wacc_range):
    for col_idx, term_growth_value in enumerate(term_growth_range):
        # 构建使用wacc_value和term_growth_value的公式
        formula = f"=<DCF重新计算使用 {wacc_value} 和 {term_growth_value}>"
        ws.cell(row=start_row+row_idx, column=start_col+col_idx).value = formula
```

**敏感性表格必须在模型打开时立即工作，无需用户手动操作。**

<common_mistakes>

本部分包含构建DCF模型时应避免的所有错误模式。

### 错误：简化敏感性表格近似或占位文本

**不要使用线性近似：**

```
// 错误 — 线性近似
B97: =B88*(1+(0.096-0.116))    // 假设线性关系

// 错误 — 除法捷径
B105: =B88/(1+(E48-0.07))      // 不重新计算完整DCF
```

**不要留占位文本：**
```
// 错误 — 占位注释
"注：使用Excel数据表功能（数据→假设分析→数据表）填充敏感性表格。"

// 错误 — 空单元格
[因"这很复杂"而留空]
```

**不要混淆术语：**
- ❌ "敏感性表格需要Excel的数据表功能"（不 — 那是我们无法使用的特定Excel工具）
- ✅ "敏感性表格是每个单元格带公式的简单网格"（是 — 这才是我们要构建的）

**为何这些捷径错误：**
- 线性近似公式不真正重新计算DCF — 只是应用简单数学调整
- 关系不是线性的，因此结果会不准确
- 占位文本需要用户手动干预
- 模型交付时不能立即使用
- 不专业，不适合客户
- 空单元格 = 未完成的交付物

**常见合理化借口 — 应拒绝：**
"写75+公式感觉复杂，所以我留注释让用户手动完成。"

**现实：** 用Python循环配合openpyxl写75个公式很直接。每个公式遵循相同模式 — 只是替换行/列值。这是交付物的必需部分。

**应改为：** 用公式填充每个敏感性单元格，对每个特定假设组合重新计算完整DCF

### 错误：缺失单元格批注

**不要这样做：**
- 创建所有硬编码输入时不加批注
- 想"我稍后再加"
- 写"TODO: 添加来源"
- 蓝色输入无文档

**为何错误：**
- 无法验证数据来源
- 违反xlsx技能要求
- 无法通过审计
- 后期修复浪费时间

**应改为：** 每个硬编码值创建时即添加批注

### 错误：公式行引用错位

**症状：**
FCF部分引用错误的假设行：
`折旧摊销: =E29*$E$34    // 应为$E$21，但引用了错误行`
`资本开支: =E29*$E$41   // 应为$E$22，但行偏移了`

**原因：**
1. 先写公式
2. 然后插入表头
3. 所有行引用偏移
4. 现在公式指向错误单元格 → #REF! 错误

**应改为：** 先锁定行布局，再写公式

### 错误：每个假设单行横跨情景

**不要这样构建假设：**
```csv
假设,悲观,基础,乐观
营收增速 FY1,8%,10%,12%
营收增速 FY2,7%,9%,11%
```
此纵向布局难以看清每个情景内假设跨年度的演进。

**为何错误：**
- 难以看清假设在每个情景内如何跨年度演进
- 难以比较不同情景在整个预测期的假设
- 审核情景逻辑时不够直观

**应改为：**
- 为每个情景（悲观、基础、乐观）创建独立区块
- 在每个区块内，横向展示预测年份的假设
- 这使每个情景的假设作为连贯集合更易于审核

### 错误：无边框

**不要交付无边框的模型：**
- 无分节划分
- 所有单元格混在一起
- 难以阅读且不专业

**为何错误：**
- 不适合客户
- 难以导航
- 显得业余

**应改为：** 为所有主要分节添加边框

### 错误：错误字体颜色或无字体颜色区分

**不要这样做：**
- 所有文字都是黑色
- 只用填充颜色（不变更字体颜色）
- 混淆哪些单元格是蓝色 vs 黑色

**为何错误：**
- 无法区分输入和公式
- 审计变得不可能
- 违反xlsx技能要求

**应改为：** 所有硬编码输入用蓝色文字，所有公式用黑色文字，跨表链接用绿色

### 错误：运营费用基于毛利润

**不要这样做：**
`销售费用: =E33*0.15    // E33 = 毛利润（错误）`

**为何错误：**
- 运营费用随营收规模变动，而非毛利润
- 产生不切实际的利润率演进
- 不符合企业实际运营方式

**应改为：**
`销售费用: =E29*0.15    // E29 = 营业收入（正确）`

### 五大错误总结

1. **公式行引用错位** → 写公式前先定义所有行位置
2. **缺失单元格批注** → 创建单元格时即添加批注，而非末尾
3. **简化敏感性表格** → 用完整DCF重新计算公式填充所有单元格，而非近似
4. **情景区块引用错误** → 确保IF公式从正确的悲观/基础/乐观区块提取
5. **无边框** → 添加专业分节边框，达到客户交付标准

此外，注意这些错误：

### WACC计算错误
- 资本结构中混用账面价值和市场价值
- 错误使用权益Beta而非资产/去杠杆Beta
- 债务成本税率应用错误
- 无风险利率错误（必须使用当前中国10Y国债收益率）
- 未针对净负债 vs 净现金头寸调整

### 增速假设缺陷
- 终值增长 > WACC（产生无穷价值）
- 预测增速与历史表现不一致
- 忽视行业增长约束
- 营收增速与单位经济不挂钩
- 无运营支撑的利润扩张

### 终值错误
- 使用错误增长法（永续增长 vs 退出倍数）
- 终值 > 企业价值80%（暗示过度依赖）
- 终值利润率与稳态假设不一致
- 终值折现期错误

### 现金流预测错误
- 运营费用基于毛利润而非营收
- 折旧摊销/资本开支百分比与业务模式不匹配
- 营运资本变动计算不当
- 各年间税率不一致
- NOPAT计算错误

**这些是最常见的错误。开始任何DCF构建前请重读本节。**

</common_mistakes>

## Excel文件创建

**本技能使用 `xlsx` 技能进行所有电子表格操作。** xlsx技能提供：
- 标准化公式构建规则
- 数字格式化约定
- 通过 `recalc.py` 脚本自动公式重算
- 全面的错误检查和验证

本技能创建的所有Excel文件必须遵循xlsx技能要求，包括零公式错误和正确重算。

## 质量评分标准

每个DCF模型必须在以下方面最大化：
1. **基于历史表现的现实营收和利润率假设**
2. **采用正确CAPM方法的适当资本成本计算**
3. **全面的敏感性分析**展示估值区间
4. **清晰的终值计算**及支撑理由
5. **支持情景分析的专业模型结构**
6. **所有关键假设的透明文档**

## 输入要求

### 最低要求输入
1. **公司标识**：股票代码或公司名称
2. **增长假设**：预测期营收增速（或"使用一致预期"）
3. **可选参数**：
   - 预测期（默认：5年）
   - 情景情形（悲观/基础/乐观增速和利润率假设）
   - 终值增长率（默认：3.0-3.5%）
   - 若不使用CAPM，提供特定WACC输入

## Excel模型结构

### 工作表架构

创建**两个工作表**：

1. **DCF** — 主估值模型，底部含敏感性分析
2. **WACC** — 资本成本计算

**关键**：敏感性表格放在DCF工作表底部（不要单独工作表）。这使所有估值输出集中在一处。

### 公式重算（强制）

创建或修改Excel模型后，**使用xlsx技能的recalc.py脚本重算所有公式**：

```bash
python recalc.py [excel文件路径] [超时秒数]
```

示例：
```bash
python recalc.py 600519_DCF_Model_2025-05-06.xlsx 30
```

脚本将：
- 使用LibreOffice重算所有工作表的所有公式
- 扫描所有单元格的Excel错误（#REF!、#DIV/0!、#VALUE!、#NAME?、#NULL!、#NUM!、#N/A）
- 返回含错误位置和数量的详细JSON

**预期输出格式：**
```json
{
  "status": "success",           // 或 "errors_found"
  "total_errors": 0,              // 错误总数
  "total_formulas": 42,           // 文件中公式数量
  "error_summary": {}             // 仅发现错误时存在
}
```

**若发现错误**，输出将包含详情：
```json
{
  "status": "errors_found",
  "total_errors": 2,
  "total_formulas": 42,
  "error_summary": {
    "#REF!": {
      "count": 2,
      "locations": ["DCF!B25", "DCF!C25"]
    }
  }
}
```

**修复所有错误**并重新运行recalc.py直到状态为 "success" 再交付模型。

### 格式标准

**重要**：遵循xlsx技能进行公式构建规则和数字格式化约定。DCF技能增加特定的视觉呈现标准。

**配色方案 — 两层：**

**第一层：字体颜色（xlsx技能强制）**
- **蓝色文字（RGB: 0,0,255）**：所有硬编码输入（股价、股本、历史数据、假设）
- **黑色文字（RGB: 0,0,0）**：所有公式和计算
- **绿色文字（RGB: 0,128,0）**：链接到其他工作表（WACC工作表引用）

**第二层：填充颜色 — 专业蓝/灰调色板（除非用户另有指定，默认使用）**
- **保持极简** — 填充只用蓝色和灰色。不要引入绿色、黄色、橙色或多重强调色。颜色过多的模型显得业余。
- **默认填充调色板：**
  - **分节标题**：深蓝（RGB: 31,78,121 / `#1F4E79`）背景白色加粗文字
  - **子标题/列标题**：浅蓝（RGB: 217,225,242 / `#D9E1F2`）背景黑色加粗文字
  - **输入单元格**：浅灰（RGB: 242,242,242 / `#F2F2F2`）背景蓝色字体 — 或若追求最大极简，仅用白色+蓝色字体
  - **计算单元格**：白色背景黑色字体
  - **输出/汇总行**（每股价值、EV等）：中蓝（RGB: 189,215,238 / `#BDD7EE`）背景黑色加粗字体
- **就这些 — 3种蓝 + 1种灰 + 白色。** 抵制添加更多颜色的冲动。
- 用户提供的模板或明确颜色偏好永远覆盖这些默认值。

**两层如何协作：**
- 输入单元格：蓝色字体 + 浅灰填充 = "硬编码输入"
- 公式单元格：黑色字体 + 白色背景 = "计算值"
- 跨表链接：绿色字体 + 白色背景 = "引用另一工作表"
- 关键输出：黑色加粗字体 + 中蓝填充 = "这是答案"

**字体颜色告诉你它是什么（输入/公式/链接）。填充颜色告诉你位于何处（表头/数据/输出）。**

### 边框标准（专业外观必需）

**粗边框**（1.5pt）围绕主要分节：
- 关键输入分节
- 预测假设分节
- 5年现金流预测分节
- 终值分节
- 估值摘要分节
- 每个敏感性分析表格

**中等边框**（1pt）在子分节之间：
- 公司详情 vs 历史表现
- 增速假设 vs EBIT利润率 vs FCF参数

**细边框**（0.5pt）围绕数据表格：
- 情景假设表格（悲观 | 基础 | 乐观 | 选定）
- 历史 vs 预测财务矩阵

**无边框：** 表格内的单个单元格（保持清洁、可扫描）

**边框是强制的** — 无边框专业边框的模型不适合客户交付。

**数字格式**（遵循xlsx技能标准）：
- **年份**：格式化为文本字符串（如 "2024" 而非 "2,024"）
- **百分比**：`0.0%`（一位小数）
- **货币**：`¥#,##0` 用于亿元；`¥#,##0.00` 用于每股 — 始终在表头中注明单位（"营业收入（亿元）"）
- **零值**：用数字格式使所有零显示为 "-"（如 `¥#,##0;(¥#,##0);-`）
- **大数字**：`#,##0` 带千分位分隔符
- **负数**：`(#,##0)` 括号（不用减号）

**单元格批注（所有硬编码输入强制）**：

按xlsx技能要求，所有硬编码值必须有记录来源的单元格批注。格式："来源：[系统/文档]，[日期]，[引用]，[URL如适用]"

**关键**：创建单元格时即添加批注。不要推迟到末尾。

**关键WACC公式：**
```
总市值 = 股价 × 总股本
净负债 = 有息负债 - 货币资金
企业价值 = 总市值 + 净负债
权益权重 = 总市值 / 企业价值
债务权重 = 净负债 / 企业价值
WACC = (权益成本 × 权益权重) + (税后债务成本 × 债务权重)
```

## 情景选择器实现

**三情景框架：**

### 悲观情形
- 保守营收增速（历史区间下限）
- 利润率压缩或无扩张
- 更高WACC（风险溢价增加）
- 更低终值增长率
- 更高资本开支假设

### 基础情形
- 一致预期或管理层指引营收增速
- 基于经营杠杆的温和利润率扩张
- 当前市场隐含WACC
- 与GDP对齐的终值增长（3.0-3.5%）
- 标准资本开支假设

### 乐观情形
- 乐观营收增速（预测上限）
- 显著利润率扩张
- 更低WACC（风险溢价降低）
- 更高终值增长（3.5-4.0%）
- 降低资本开支强度

**公式实现：**

**不要在全文中分散使用嵌套IF公式。** 而是创建使用INDEX或OFFSET公式从适当情景区块提取的合并列。

**推荐模式（使用INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`，B10:D10 = 悲观/基础/乐观值，1 = 行偏移，$B$6 = 情景选择单元格（1、2或3）

**然后在所有预测中引用合并列：**
`营收第1年: =D29*(1+$E$10)`，$E$10为第1年增速合并列值。

此方法集中情景逻辑，使模型更易于审计和维护。

## 常见变体

### 高成长科技公司
- 更长预测期（7-10年）
- 更高初始增速（20-30%）
- 随时间显著利润率扩张
- 更高WACC（12-15%）
- 建模单位经济（用户数、ARPU等）

### 成熟/稳定公司
- 更短预测期（3-5年）
- 温和增速（GDP +1-3%）
- 稳定利润率
- 更低WACC（7-9%）
- 聚焦现金创造和资本配置

### 周期性公司
- 穿越经济周期建模
- 中周期利润率正常化
- 考虑低谷和高峰情景
- 针对周期性调整Beta

### 多业务板块公司
- 每个业务单元单独DCF
- 各板块不同增速和利润率
- 分部加总估值
- 考虑协同效应
